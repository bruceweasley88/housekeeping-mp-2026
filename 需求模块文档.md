# 需求模块文档

## 一、模块概述

需求模块是家政小程序的核心功能，支持用户发布各类需求，其他用户可以承接并完成需求。

### 需求分类

1. 轻需求
2. 接送需求
3. 维修需求
4. 陪伴需求
5. 物品交易
6. 家政服务
7. 其他需求

### 业务流程

```
发布需求 → 待接单 → 承接需求 → 进行中 → 确认完成 → 已完成 → 平台结算 → 已结算
                                                    ↓
                                               取消承接（回到待接单）
```

### 状态说明

| 状态值 | 状态名 | 说明 |
|--------|--------|------|
| 1 | 待接单 | 需求已发布，等待承接 |
| 2 | 进行中 | 已被承接，正在进行 |
| 3 | 已完成 | 发布者确认完成，待结算 |
| 4 | 已取消 | 需求被取消 |
| 5 | 已下架 | 需求被下架 |
| 6 | 已结算 | 平台结算完成 |

- 同一需求可被多次承接（承接→取消→再承接），每次产生新记录

- 查询当前承接状态时，取该需求最后一条记录判断
- 承接者取消后，可再次承接同一需求（产生新记录）

## 三、业务逻辑

### 1. 发布需求

1. 生成需求编号：DQ + 年月日 + 4位序号
2. 根据金额类型计算金额：
   - 按小时：金额 = 每小时金额 × 总小时
   - 按次数：金额 = 金额字段
   - 按范围：金额 = 最大金额
3. 如果紧急发布，设置服务费率3%
4. 初始状态为"待接单"

### 2. 承接需求

1. 验证需求状态为"待接单"
2. 验证不能承接自己发布的需求
3. 更新需求状态为"进行中"
4. 记录承接用户ID和承接时间
5. 创建承接记录

### 3. 取消承接

1. 验证需求状态为"进行中"
2. 验证当前用户是承接者
3. 更新需求状态为"待接单"
4. 清空承接用户ID和承接时间
5. 更新承接记录状态为"已取消"

### 4. 确认完成

1. 验证需求状态为"进行中"
2. 验证当前用户是发布者
3. 更新需求状态为"已完成"

### 5. 调整金额

1. 验证当前用户是发布者
2. 验证需求状态为"进行中"或"已完成"
3. 更新**金额**字段

### 6. 平台结算（支付时）

1. 计算服务费 = 金额 × 服务费率（紧急发布3%，否则0%）
2. 计算最终结算金额 = 金额 + 服务费
3. 发起微信支付
4. 支付成功后记录最终结算金额、服务费、结算时间

## 五、调整金额弹窗

### 功能

发布者在需求"进行中"或"已完成"时，可调整**金额**字段。

### 弹窗内容

- 标题："调整金额"
- 输入框：单个金额输入（仅数字，保留2位小数）
- 当前金额显示：显示当前金额供参考
- 按钮：取消、确定

### 流程

1. 点击"调整金额"按钮
2. 弹出输入框，默认值为当前金额
3. 输入新金额
4. 点击确定 → 调用调整金额接口 → 更新金额字段 → 成功后刷新页面

## 六、价格弹窗（发布需求时使用）

### 三种类型

| 类型 | 输入字段 | 显示格式 |
|------|----------|----------|
| 按小时 | 总小时 + 每小时金额 | ¥80/小时 × 2小时 |
| 按次数 | 金额 | ¥50/次 |
| 按范围 | 最小金额 + 最大金额 | ¥150 ~ ¥300 |

### 类型切换

- 顶部三个Tab：按小时收费、按次数收费、按范围填写
- 点击切换，显示对应的输入框

### 金额计算

- 按小时：金额 = 每小时金额 × 总小时
- 按次数：金额 = 输入的金额
- 按范围：金额 = 最大金额
