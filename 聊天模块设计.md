# 聊天模块设计文档

## 1. 功能概述

一个简单的一对一用户聊天系统，采用轮询方式实现实时通信。

### 1.1 核心特性
- 仅支持用户对用户的一对一聊天
- 仅支持文字消息
- 采用1秒轮询获取最新消息
- 会话列表显示未读数量和最后消息预览
- 进入对话后自动清除未读数量

---

## 2. 数据库设计

### 2.1 会话表 `ls_chat_session`

| 字段 | 类型 | 说明 |
|------|------|------|
| id | int | 主键，自增 |
| user1_id | int | 用户1的ID（较小的用户ID） |
| user2_id | int | 用户2的ID（较大的用户ID） |
| create_time | int | 创建时间戳 |
| update_time | int | 最后更新时间戳 |

**设计说明：**
- `user1_id` 始终存储较小的用户ID，`user2_id` 存储较大的用户ID
- 通过 `(user1_id, user2_id)` 唯一索引保证两个用户之间只有一个会话
- 不冗余存储最后消息，查询会话列表时关联消息表获取

### 2.2 消息表 `ls_chat_message`

| 字段 | 类型 | 说明 |
|------|------|------|
| id | int | 主键，自增 |
| session_id | int | 会话ID |
| sender_id | int | 发送者用户ID |
| receiver_id | int | 接收者用户ID |
| content | text | 消息内容（文字） |
| is_read | tinyint | 是否已读：0-未读，1-已读 |
| create_time | int | 创建时间戳 |

**索引设计：**
- 主键索引：`id`
- 会话索引：`session_id`
- 组合索引：`session_id, id`（用于分页查询）

## 3. 业务逻辑设计

### 3.1 发送消息流程

```
用户A发送消息给用户B
    ↓
1. 查找或创建会话（user1_id=min(A,B), user2_id=max(A,B)）
    ↓
2. 创建消息记录（is_read=0）
    ↓
3. 更新会话的 update_time
    ↓
4. 返回消息ID和创建时间
```

### 3.2 获取会话列表流程

```
用户请求会话列表
    ↓
1. 查询用户参与的所有会话
    ↓
2. 关联消息表获取每个会话的最后一条消息
    ↓
3. 统计每个会话的未读消息数（receiver_id=当前用户 AND is_read=0）
    ↓
4. 按 update_time 倒序排序
    ↓
5. 返回列表
```

### 3.3 获取对话消息流程

```
用户进入对话界面
    ↓
1. 分页获取消息（每页20条，按id倒序）
    ↓
2. 如果是首次进入（last_id=0），将该会话中接收的消息标记为已读
    ↓
3. 支持增量获取（传入last_message_id，只返回id更大的消息）
    ↓
4. 返回消息列表
```

### 3.4 清除未读数

```
用户进入对话界面时
    ↓
1. 查询当前会话中，receiver_id=当前用户 AND is_read=0 的消息
    ↓
2. 批量更新为 is_read=1
    ↓
3. 返回成功
```

## 4. API 设计

### 4.1 会话相关 API

#### 4.1.1 获取会话列表
- **URL**: `GET /api/chat/session/list`
- **说明**: 获取当前用户的所有会话列表
- **响应**:
```json
{
  "code": 1,
  "msg": "success",
  "data": {
    "list": [
      {
        "session_id": 1,
        "peer_user": {
          "id": 2,
          "nickname": "张三",
          "avatar": "https://..."
        },
        "last_message": {
          "content": "你好",
          "create_time": 1711084800
        },
        "unread_count": 3,
        "update_time": 1711084800
      }
    ]
  }
}
```

#### 4.1.2 创建/获取会话
- **URL**: `POST /api/chat/session/create`
- **说明**: 与指定用户创建或获取已有会话
- **参数**:
```json
{
  "user_id": 2
}
```
- **响应**:
```json
{
  "code": 1,
  "msg": "success",
  "data": {
    "session_id": 1
  }
}
```

### 4.2 消息相关 API

#### 4.2.1 发送消息
- **URL**: `POST /api/chat/message/send`
- **说明**: 发送文字消息
- **参数**:
```json
{
  "session_id": 1,
  "content": "你好"
}
```
- **响应**:
```json
{
  "code": 1,
  "msg": "success",
  "data": {
    "message_id": 100,
    "create_time": 1711084800
  }
}
```

#### 4.2.2 获取消息列表
- **URL**: `GET /api/chat/message/list`
- **说明**: 分页获取会话消息，支持增量获取
- **参数**:
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| session_id | int | 是 | 会话ID |
| last_id | int | 否 | 上次获取的最后消息ID，用于增量获取（0或不传表示首次加载） |
| page | int | 否 | 页码，默认1（仅在首次加载时有效） |
| page_size | int | 否 | 每页数量，默认20 |

- **响应**:
```json
{
  "code": 1,
  "msg": "success",
  "data": {
    "list": [
      {
        "id": 100,
        "sender_id": 1,
        "content": "你好",
        "is_self": false,
        "create_time": 1711084800,
        "create_time_text": "刚刚"
      }
    ],
    "has_more": true
  }
}
```

#### 4.2.3 清除未读数
- **URL**: `POST /api/chat/message/read`
- **说明**: 将会话中接收的消息标记为已读
- **参数**:
```json
{
  "session_id": 1
}
```
- **响应**:
```json
{
  "code": 1,
  "msg": "success"
}
```

## 5. 前端实现要点

### 5.1 轮询策略

#### 会话列表页面
- 每1秒轮询 `/api/chat/session/list`
- 显示未读总数（所有会话未读数之和）

#### 对话界面
- 每1秒轮询 `/api/chat/message/list`（使用 last_id 增量获取）
- 进入页面时调用 `/api/chat/message/read` 清除未读
- 发送消息后立即刷新消息列表

### 5.2 消息列表滚动加载
- 首次加载获取最新20条消息
- 向上滚动到顶部时，加载更早的消息（page + 1）
- 新消息通过轮询增量添加到底部

### 5.3 智能时间显示
- 1分钟内：刚刚
- 1小时内：X分钟前
- 今天：HH:mm
- 昨天：昨天 HH:mm
- 今年：MM-DD HH:mm
- 往年：YYYY-MM-DD HH:mm
